Create a new file `nginx/nginx.conf` that replaces Ubuntu's default `/etc/nginx/nginx.conf`. This file manages global settings that the current project does not configure at all (Ubuntu defaults are in use).

The file must contain these exact blocks:

**Top-level:**
```
user www-data;
worker_processes auto;
pid /run/nginx.pid;
worker_rlimit_nofile 8192;
include /etc/nginx/modules-enabled/*.conf;
```

`worker_processes auto` resolves to 2 on the t4g.small (2 vCPU). `worker_rlimit_nofile 8192` raises the per-worker file descriptor limit. (INFRA-05)

**events block:**
```
events {
    worker_connections 1024;
    multi_accept on;
    use epoll;
}
```

1024 connections per worker x 2 workers = 2048 max concurrent. `epoll` is the Linux-optimal event model. (INFRA-05)

**http block — gzip (INFRA-01):**
```
gzip on;
gzip_comp_level 5;
gzip_min_length 256;
gzip_vary on;
gzip_proxied any;
gzip_types
    text/plain
    text/css
    text/javascript
    application/javascript
    application/json
    application/xml
    image/svg+xml;
```

Level 5 (not 6) because ARM64 has slower per-core compression — 95% of compression benefit at 15% less CPU. `gzip_min_length 256` avoids compressing tiny responses. Do NOT include `application/pdf` (PDFs are already binary-compressed). `gzip_proxied any` ensures gzip works for proxied responses from Anubis and backends.

**http block — TLS session cache (INFRA-02):**
```
ssl_session_cache shared:SSL:10m;
ssl_session_timeout 10m;
```

Caches ~40,000 TLS sessions shared across workers. Repeat visitors skip full TLS handshake (~200-400ms savings).

**http block — HTTP/2 (INFRA-03):**

Do NOT add `http2 on` in nginx.conf — it must go in each `listen 443 ssl` server block in the site config. However, HTTP/2 requires the site config to add it. For this task, add a comment in nginx.conf:
```
# HTTP/2: enabled per-server in sites-enabled configs via "http2 on;" directive
```

The actual `http2 on;` directive will be added to `asanchezbl.dev.conf` in Task 2.

**http block — performance:**
```
sendfile on;
tcp_nopush on;
tcp_nodelay on;
types_hash_max_size 2048;
server_tokens off;

keepalive_timeout 65s;
keepalive_requests 100;
```

`keepalive_requests 100` (not the default 1000) recycles connections faster on 2GB RAM, freeing per-connection allocations.

**http block — includes (CRITICAL — must match Ubuntu layout):**
```
include /etc/nginx/mime.types;
default_type application/octet-stream;

access_log /var/log/nginx/access.log;
error_log /var/log/nginx/error.log;

include /etc/nginx/conf.d/*.conf;
include /etc/nginx/sites-enabled/*;
```

The `include` directives load the site config from `sites-enabled/` which is the existing deploy path.

**Part A: Add HTTP/2 to site config (INFRA-03)**

In `nginx/conf.d/asanchezbl.dev.conf`, add `http2 on;` to the main HTTPS server block (the one with `server_name asanchezbl.dev;`). Place it immediately after the `listen [::]:443 ssl;` line:

```nginx
listen 443 ssl;
listen [::]:443 ssl;
http2 on;
server_name asanchezbl.dev;
```

Also add `http2 on;` to the www-redirect HTTPS server block:

```nginx
listen 443 ssl;
listen [::]:443 ssl;
http2 on;
server_name www.asanchezbl.dev;
```

Do NOT add `http2 on;` to the internal `listen 127.0.0.1:8080;` server block (localhost traffic does not benefit from HTTP/2 and it does not use TLS).

**CRITICAL for sub_filter compatibility:** In the internal server block (`:8080`), the demo locations (`/demo/serp/` and `/demo/catlink/`) already have `proxy_set_header Accept-Encoding "";` which prevents gzip from breaking sub_filter. This is correct. Do NOT remove those lines. The gzip directives in nginx.conf apply globally but the `Accept-Encoding ""` header strips compression for those specific proxied requests, preserving sub_filter functionality. See STATE.md blocker about this interaction.

**Part B: Update Ansible nginx role to deploy nginx.conf**

In `ansible/roles/nginx/tasks/main.yml`, add a new task BEFORE the existing "Copy Nginx site config" task:

```yaml
- name: Deploy global nginx.conf
  copy:
    src: "{{ playbook_dir }}/../../nginx/nginx.conf"
    dest: /etc/nginx/nginx.conf
    owner: root
    group: root
    mode: '0644'
    backup: yes
  notify: reload nginx
```

The `backup: yes` preserves the old nginx.conf on the server as a `.bak` file.

**Part C: Add nginx -t validation to handler**

Replace the handler in `ansible/roles/nginx/handlers/main.yml` with a two-step handler that validates before reloading:

```yaml
---
- name: validate nginx config
  command: nginx -t
  changed_when: false
  listen: "reload nginx"

- name: reload nginx
  systemd:
    name: nginx
    state: reloaded
  listen: "reload nginx"
```

This uses Ansible's `listen` directive so both handlers fire on the "reload nginx" notification. The validate step runs first (handlers fire in definition order). If `nginx -t` fails, the playbook aborts before the reload happens, keeping the old config active. This prevents Pitfall 1 (config syntax error kills entire site).
