upstream anubis {
    server 127.0.0.1:8923;
    keepalive 16;

upstream cv_service {
    server 127.0.0.1:8001;
    keepalive 4;
}

upstream serp_frontend {
    server 127.0.0.1:3001;
    keepalive 4;

Create a new Ansible role `swap` that provisions a 2GB swap file on the EC2 instance. (INFRA-06)

**Step 1: Create the role directory structure:**

Create `ansible/roles/swap/tasks/main.yml` with these tasks:

```yaml
---
- name: Check if swap file exists
  stat:
    path: /swapfile
  register: swap_file

- name: Create 2GB swap file
  command: fallocate -l 2G /swapfile
  when: not swap_file.stat.exists

- name: Set swap file permissions
  file:
    path: /swapfile
    mode: '0600'
    owner: root
    group: root
  when: not swap_file.stat.exists

- name: Format swap file
  command: mkswap /swapfile
  when: not swap_file.stat.exists

- name: Enable swap file
  command: swapon /swapfile
  when: not swap_file.stat.exists

- name: Add swap to fstab for persistence
  lineinfile:
    path: /etc/fstab
    line: '/swapfile none swap sw 0 0'
    state: present

- name: Set swappiness to 10 (prefer RAM, use swap only under pressure)
  sysctl:
    name: vm.swappiness
    value: '10'
    state: present
    sysctl_set: yes
    reload: yes
```

Why swappiness 10: Default is 60, which is too aggressive for a server — the kernel would swap out application pages even with free RAM available. Value 10 means the kernel only uses swap under real memory pressure (during Docker image pulls, WeasyPrint startup peaks). This keeps normal operation fully in RAM while preventing OOM kills during deploy spikes.

All tasks use `when: not swap_file.stat.exists` for idempotency — re-running the playbook does not recreate or re-format an existing swap file.

**Step 2: Add swap role to site.yml playbook**

In `ansible/playbooks/site.yml`, add the swap role in the "Base system setup" play, AFTER the `base` role and BEFORE the `docker` role:

```yaml
- name: Base system setup
  hosts: portfolio
  become: true
  roles:
    - role: base
      tags: base
    - role: swap
      tags: swap
    - role: security
      tags: security
    - role: docker
      tags: docker
    - role: nginx
      tags: nginx
    - role: certbot
      tags: certbot
```

The swap role must run before Docker because Docker image pulls are the primary OOM risk that swap mitigates (see Pitfall 5 in PITFALLS.md). Running it after base ensures apt packages are up to date first.
